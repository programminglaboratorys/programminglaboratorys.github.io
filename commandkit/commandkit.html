commandkit.parser.eval_args(expected_args: list, args: list, allow_overflow: bool = False, missing_okay: bool = True, missing: Any = Missing) → tuple[dict[str, str | list[str]], list[str]][source]

A function to simulate python functions positional/var positional arguments.

Note: it modify args argument

Parameters:

  • expected_args (list) – the format of the arguments

  • args (list) – arguments

  • allow_overflow (bool (default: False)) – whether to allow extra arguments

  • missing_okay (bool (default: True)) – whether to allow missing arguments

  • missing (Any (default: Missing)) – the value to use for missing arguments

Raises:

  • ValueError – if a star was provided but not followed by an argument name

  • MissingError – if a required argument is missing

  • EAOverflowError – if too many arguments are provided

Returns:

tuple of two elements; the arguments, the remaining arguments

Return type:

tuple[dict[str, str | list[str]], list[str]]

Example

>> eval_args(["whatup","hmm","*","hehe"],"good ye? more stuff :)".split())
({'whatup': 'good', 'hmm': 'ye?', 'hehe': ['more', 'stuff', ':)']}, [])
